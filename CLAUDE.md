# プロジェクトルール

## テスト

機能の追加・変更を行う際は、必ず以下のテストを伴わせること。

1. **単体テストを作成すること**
   - 新しいロジック(`src/lib` など)を追加・変更したら、対応する単体テストを `tests/unit/` に追加する。
   - 単体テストは古典学派(Classicist)の方針で書く:
     - 可能な限り本物のオブジェクトを使う(モック/スタブは持ち込まない)。
     - 検証するのは「状態」や「結果」。
     - テスト対象は「振る舞いの単位」(必ずしも 1 関数・1 クラスではない)。
   - 実行: `npm test`

2. **フロントエンドにも関わる部分は、E2E テストに動作確認を追加すること**
   - 画面の挙動(UI からの操作・表示)に影響する変更は、`tests/e2e/` に動作確認を追加する。
   - 本物のアプリをブラウザから操作し、結果を検証する。
   - 実行: `npm run test:e2e`

詳細な実行手順は [README.md](README.md) の「テスト」節を参照。
